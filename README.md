![Spring Boot](https://img.shields.io/badge/Spring_Boot_3-6DB33F?style=flat&logo=springboot&logoColor=white)
![Spring Security](https://img.shields.io/badge/Spring_Security_6+JWT-6DB33F?style=flat&logo=springsecurity&logoColor=white)
![React](https://img.shields.io/badge/React_18-61DAFB?style=flat&logo=react&logoColor=black)
![MySQL](https://img.shields.io/badge/MySQL-4479A1?style=flat&logo=mysql&logoColor=white)
![Docker](https://img.shields.io/badge/Docker-2496ED?style=flat&logo=docker&logoColor=white)
![JUnit5](https://img.shields.io/badge/JUnit_5-25A162?style=flat&logo=junit5&logoColor=white)
![Swagger](https://img.shields.io/badge/Swagger_UI-85EA2D?style=flat&logo=swagger&logoColor=black)

A production-grade full-stack web portal with **Admin & Student RBAC** — **9 REST controllers**, **15+ secured endpoints**, React 18 frontend, MySQL, Docker, JUnit 5 tests, and full Swagger UI / OpenAPI documentation.

<details>
<summary><b>📋 Key Technical Highlights</b></summary>
<br/>

**🔐 Security & Auth**
- Stateless JWT authentication — login, forced first-login password change with **short-lived 15-min tokens**
- `@PreAuthorize` method-level RBAC; JwtFilter for token validation; auto-seeded default admin via `DataSeeder`
- `POST /auth/register` is **ADMIN-only** — no public admin creation vulnerability

**🏛️ Admin Panel**
- Student CRUD with live name/email filtering and **inline edit row**
- Course, subject, and marks management; subjects load dynamically by department
- **Duplicate-safe daily attendance** marking with department filter and real-time counters
- Analytics dashboard with **Chart.js** bar charts and live stat counters

**👨‍🎓 Student Panel**
- Personalised dashboard: average marks, animated SVG attendance ring chart, recent announcements
- Grade letters (A+→D) with progress bars; duplicate-safe course enrollment
- Private routes with role-based routing; **Axios interceptor for auto-logout on 401**

**⚙️ Backend Architecture**
- **Cascade delete** — removes marks, attendance, enrollment & login account atomically via `@Transactional`
- Duplicate prevention at **3 levels**: DB unique constraints, service-layer checks, `GlobalExceptionHandler`
- `DataIntegrityViolationException` → 409 Conflict with clear error messages
- Containerised with **Docker**; all APIs documented with **Swagger UI / OpenAPI**; **JUnit 5** unit tests

</details>

### 🩺 Diabetes Disease Prediction System
> **Nov 2024 – Feb 2025**

[![GitHub Repo](https://img.shields.io/badge/GitHub_Repo-181717?style=for-the-badge&logo=github)](https://github.com/Maralishashank/Diabetes-Disease-Prediction)

![Python](https://img.shields.io/badge/Python-3776AB?style=flat&logo=python&logoColor=white)
![Django](https://img.shields.io/badge/Django-092E20?style=flat&logo=django&logoColor=white)

![Pandas](https://img.shields.io/badge/Pandas-150458?style=flat&logo=pandas&logoColor=white)

An ML-powered healthcare web app for early diabetes risk prediction using the **Pima Indians Diabetes Dataset (768 records)**.

- 📊 EDA on 768 records with Pandas and Matplotlib — preprocessing, feature analysis
- 🤖 Implemented and evaluated **5 ML classification algorithms** (KNN, Naïve Bayes, Decision Tree, Random Forest, SVM)
- 🎯 **KNN achieved 76% prediction accuracy** — best performing model with cross-validation
- 🌐 Django web interface for real-time user input → instant, interpretable predictions
